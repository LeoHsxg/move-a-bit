# 動一下 App — Wireframe 規格書
> 給 Claude Design 的完整設計指引

---

## 一、產品概覽

**App 名稱**：動一下  
**平台**：iOS（iPhone，以 390×844pt 為基準，iPhone 14 尺寸）  
**設計風格**：Minimalist、溫暖低壓、去焦慮  
**核心主張**：
- 不評斷運動量，只慶祝每一次「有出現」
- 不要求連續打卡，只累積每一個印記
- 不製造身材焦慮，只提供陪伴與意義感

**色彩方向**：柔和米白底、主色建議暖色系（米黃、淡橘、或淡綠），避免高飽和度強烈配色。無紅色警示類元素。

---

## 二、整體架構

```
動一下
├── Onboarding（首次使用，共 5 頁）
│   ├── 01 歡迎畫面
│   ├── 02 選擇運動大目標
│   ├── 03 設定容易運動的時間
│   ├── 04 設定預設運動地點
│   └── 05 好友分享偏好
│
└── 主 App（4 個 Tab）
    ├── 首頁 Tab — 每日任務卡片
    ├── 紀錄 Tab — 日曆印記 + 影片回顧
    ├── 好友 Tab — 好友動態 + 戳一戳
    └── 我的 Tab — 目標 / 稱號 / 設定
```

## 三、Onboarding 流程（5 個畫面）

> 設計原則：輕鬆、溫暖、不填表格感。每頁只做一件事。進度條顯示在頂部（1/5、2/5…）。

---

### Onboarding 01 — 歡迎畫面

**版面：**
- 全螢幕背景（柔和米白或溫暖漸層）
- 中央大字：**「動一下」**（App 名稱，大且有溫度）
- 副標一行：「每天只要動一下，就夠了」
- 底部：大型 CTA 按鈕「開始」

**元素：**
- 無 logo 圖案要求，文字即主視覺
- 無 Skip 按鈕

---

### Onboarding 02 — 選擇運動大目標

**版面：**
- 頂部進度條（2/5）
- 標題：「你最想要的是什麼？」
- 副標：「不用很精確，選一個最有共鳴的就好」
- 三張卡片（垂直排列或 2+1 排列）：
  - 卡片 A：**變健康**（副文字：「讓身體好一點」）
  - 卡片 B：**改善體態**（副文字：「讓自己看起來更喜歡」）
  - 卡片 C：**找回照顧自己的感覺**（副文字：「重新掌控生活節奏」）
- 卡片為單選，選中後有明顯高亮狀態
- 底部：「下一步」按鈕（選擇後才啟用）

**設計注意：**
- 卡片文字不出現「減肥」「瘦身」「體重」等字眼
- 卡片要有溫度，不像選項表格

---

### Onboarding 03 — 設定容易運動的時間

**版面：**
- 頂部進度條（3/5）
- 標題：「你平常什麼時候比較容易動？」
- 副標：「可以複選，之後可以改」
- 三個時間選項（Chip / Tag 樣式，可複選）：
  - 起床後上課前
  - 下課後（回宿舍前）
  - 晚飯後
- 選中有填色高亮，未選為邊框樣式
- 底部：「下一步」按鈕

---

### Onboarding 04 — 設定預設運動地點

**版面：**
- 頂部進度條（4/5）
- 標題：「你最常在哪裡動？」
- 副標：「App 靠近這裡時會輕輕提醒你」
- 地點選項（單選，卡片或列表樣式）：
  - 學校操場 / 運動場
  - 學校健身房
  - 附近公園
  - 宿舍地板（居家）
  - 自己輸入（文字欄位）
- 底部：「下一步」按鈕 + 小字「之後可以在設定裡更改」

---

### Onboarding 05 — 好友分享偏好

**版面：**
- 頂部進度條（5/5）
- 標題：「要讓朋友看到你的進度嗎？」
- 副標：「只有你邀請的朋友才能看到，你也可以看到對方的」
- 兩個選項（大型 Radio Button 卡片）：
  - **開放好友查看** — 「互相知道彼此有沒有在動，低壓陪伴」
  - **先保持私人** — 「之後可以在設定裡開啟」
- 底部：「完成，開始動一下」按鈕（主要 CTA）

## 四、主 App 畫面

### 全域元素

**Bottom Tab Bar（4 個 Tab）：**

| Icon | 標籤 |
|------|------|
| 🏠 房子 | 首頁 |
| 📅 日曆 | 紀錄 |
| 👥 人群 | 好友 |
| 👤 人像 | 我的 |

Tab Bar 背景為白色或米白，有細分隔線。Active Tab 用主色標示。

---

### 首頁 Tab

**整體設計目標：** 打開即看到今天要做什麼，一個按鈕就能開始。資訊極度精簡。

**版面（由上至下）：**

1. **頂部問候區**
   - 左側：小文字時段問候（「早安」/ 「下課了」/ 「吃完飯了嗎」，根據時間自動切換）
   - 右側：累計印記數字 + 印記 icon（例如「🌿 12」）

2. **今日任務卡片**（畫面最主要元素，佔畫面約 50%）
   - 大型圓角卡片，背景用柔和暖色
   - 卡片頂部小標：「今天的任務」
   - 任務名稱（大字）：例如「去操場走走」/ 「伸展 5 分鐘」/ 「散步 10 分鐘」
   - 任務說明（小字）：例如「走到操場待一下就算完成」
   - 卡片底部：**「開始」** 大型主要按鈕

3. **本週小進度**（任務卡片下方，簡單一行）
   - 「這週已經動了 X 次」
   - 用小點點或簡單圖示表示（不用數字條、不顯示目標上限）

4. **底部 Tab Bar**

**空狀態（第一次使用）：**
- 任務卡片顯示第一個任務，文案更溫暖：「第一次？走到操場看看就好 🌱」

---

### 任務執行流程（從首頁開始，共 3 個畫面）

> 這是 App 最核心的互動流程。

---

#### 任務流程 01 — 計時畫面

**觸發：** 使用者點擊首頁「開始」按鈕

**版面：**
- 全螢幕，沉浸式，隱藏 Tab Bar
- 頂部左側：「✕」關閉（離開計時，需確認）
- 中央：
  - 任務名稱（中型字）
  - 大型倒數計時數字（例如「05:00」，以分：秒顯示）
  - 計時圓弧進度環繞在數字外圍（可選）
- 底部：
  - **「我完成了」** 按鈕（主要，在計時進行中也可點擊，提前完成）
  - 小字：「或等計時結束」

**計時結束後：** 自動進入下一個畫面（相機畫面）

---

#### 任務流程 02 — 3 秒相機畫面

**觸發：** 計時結束 或 點擊「我完成了」

**版面設計：** 類似復古即可拍 / Polaroid 相機介面風格

- 整個畫面以相機取景框為主視覺
- 取景框頂部的引導問句（問句語氣，非指令）：
  - 「記錄一下今天的樣子？」
- 底部操作區：
  - 中間：**圓形快門按鈕**（大）— 錄製 3 秒影片
  - 右側：**「跳過」**（小字連結，不明顯但存在）
- 按下快門後：錄製 3 秒 → 自動進入完成畫面

**設計注意：**
- 相機介面要有溫度，不要像監控鏡頭
- 可考慮加入柔和的暈邊效果（類 Polaroid 感）
- 跳過不需要確認步驟，直接跳轉

---

#### 任務流程 03 — 完成 / 慶祝畫面

**觸發：** 拍完影片 或 點擊跳過

**版面：**
- 全螢幕，有輕微慶祝視覺效果（柔和的粒子動畫或漣漪，非爆炸煙火）
- 中央大字（主訊息）：
  > **你今天有出現。**
- 中央中字（次要訊息）：
  > 「這是你的第 **12** 個印記」（數字大且突出）
- 底部：**「回首頁」** 按鈕

**設計注意：**
- 不顯示「你跑了多少公里」「燃燒了多少卡」等數字
- 文案只強調「出現」本身，不評斷運動量
- 慶祝效果溫和，不要過於喧鬧

---

### 紀錄 Tab

**設計目標：** 讓使用者看見自己「出現過的日子」，強化累積感，不強調缺席。

**版面（由上至下）：**

1. **頁面標題**：「我的印記」

2. **月曆區塊**
   - 標準月曆格式（週一至週日）
   - 有印記的日期：顯示一個小圓點或小葉子 icon（主色）
   - 沒有印記的日期：**空白，不標記、不灰色、不顯示 ✕**（關鍵：不強調缺席）
   - 今天：細邊框標示

3. **影片回顧區**（月曆下方）
   - 標題：「最近的樣子」
   - 橫向捲動的影片縮圖列表
   - 每個縮圖顯示日期小標
   - 沒有影片的日子不出現在列表中
   - 最右側：「查看全部」

4. **本月小結**（影片列表下方，選配）
   - 「這個月你出現了 X 次」（只顯示正向數字）

---

### 好友 Tab

**設計目標：** 低壓陪伴感，看到朋友有在動，可以輕輕提醒對方。無競爭、無排名。

**版面（由上至下）：**

1. **頁面標題**：「一起動」

2. **好友列表**
   - 每個好友一列，包含：
     - 頭像（圓形）
     - 名字
     - 狀態：「今天有動 🌿」/ 「3 天沒出現了」（溫和語氣，非責備）
     - 右側：**「戳一下」** 按鈕（Outline 樣式小按鈕）
   - 點擊「戳一下」後：按鈕變為灰色「已戳」，不可重複；對方收到通知

3. **底部：邀請好友**
   - 「邀請朋友一起動」連結/按鈕

**戳一下通知（對方收到）：**
- 通知文字：「[朋友名字] 戳了你一下，今天要動嗎？」

---

### 我的 Tab

**設計目標：** 個人資料、查看稱號、調整設定。乾淨，不放數字排行。

**版面（由上至下）：**

1. **個人區塊**
   - 頭像 + 名字
   - 累計印記總數：**「共 XX 個印記」**（大字，突出）
   - 目前稱號（徽章 icon + 稱號名稱，例如「常出現的人 🌱」）

2. **稱號展示區**
   - 橫向捲動卡片，顯示已解鎖和待解鎖的稱號
   - 已解鎖：彩色顯示
   - 未解鎖：灰色 + 解鎖條件（例如「再出現 5 次就能解鎖」）
   - 稱號設計方向（文字為主，不走競技感）：
     - 第 1 次：「第一步 🌱」
     - 第 5 次：「開始習慣了」
     - 第 10 次：「常出現的人」
     - 第 20 次：「有在照顧自己」
     - 第 50 次：「這已經是你的一部分了」

3. **目標設定**（可點擊進入修改）
   - 「我的目標：找回照顧自己的感覺」+ 編輯 icon

4. **設定列表**
   - 容易運動的時間 ›
   - 預設運動地點 ›
   - 好友可見性 ›
   - 通知設定 ›

## 五、其他互動細節

### 地點接近通知

- 使用者接近預設運動地點（距離約 500m 內）時
- 推送通知文字（問句，非命令）：
  > 「你現在離操場很近，要去動 5 分鐘嗎？」
- 通知有兩個快捷動作按鈕：**「出發」** / **「等等再說」**

### 月回顧

- 每月 1 日自動生成上個月的回顧
- 若有 3 秒影片，自動剪接成約 30 秒的回顧影片
- 若無影片，顯示印記統計卡片（例如：「上個月你出現了 8 次 ✨」）
- 可選擇分享（僅月回顧，非每日打卡）

## 六、設計禁止事項

在整個 App 中，**以下元素禁止出現**：

| 禁止項目 | 原因 |
|----------|------|
| 連續天數（Streak）倒數 | 中斷後會造成挫折感 |
| 卡路里 / 燃脂數字 | 觸發外貌焦慮 |
| 體重 / BMI 輸入欄位 | 不符合去焦慮設計原則 |
| 排行榜 / 競爭元素 | 造成社會比較壓力 |
| 缺席日期的紅色標記 | 強化失敗感 |
| 「你已經 X 天沒運動了」類提示 | 負面情緒驅動 |
| 健身網紅圖片 / 完美體態照片 | 觸發 SPA 社會體型焦慮 |
